# RubberDuck PRD (Product Requirements Document)

> 본 문서는 **RubberDuck** 앱의 목적·범위·요구사항에 대한 **진실의 출처(Source of Truth)** 입니다.
> AI 심사 에이전트 및 개발자는 제품의 의도를 판단할 때 이 문서를 최우선 기준으로 사용합니다.
> 구현 세부는 [SPEC.md](SPEC.md), 진행 상황은 [ROADMAP.md](ROADMAP.md), 데이터/API는 [docs/design.md](docs/design.md)를 참고합니다.

---

## 1. 제품 한 줄 정의

할 일을 **아이젠하워 매트릭스(중요도 × 긴급도)** 로 정리하고, 화면의 **3D 러버덕("표독비서")** 이
사용자의 행동에 **까칠하게(츤데레식 팩폭)** 반응하며 미루는 습관을 깨고 집중을 돕는 웹 앱.

## 2. 문제 정의 (Why)

- 할 일이 많은 사용자는 **무엇부터 할지** 우선순위를 정하기 어렵다.
- 일반적인 할 일 앱은 동기를 주지 못하고, **미루기(procrastination)** 를 막지 못한다.
- 위로형 마스코트는 식상하다. **재미·자극 요소**가 있어야 실제 행동으로 이어진다.

## 3. 솔루션 개요 (What)

1. **아이젠하워 매트릭스**로 우선순위를 시각적으로 정리한다.
2. **표독비서 러버덕**이 미루는 행동을 능청스럽게 비꼬며 행동을 유도한다.
3. **맥락(기분·에너지·날씨 등)** 을 반영해 무엇을 먼저 할지 보조 추천한다.

## 4. 목표 사용자 (Who)

- 할 일이 많아 우선순위 정리가 필요한 사람
- 미루는 습관이 있어 가벼운 자극·재미 요소가 필요한 사람

## 5. 제품 컨셉 — "표독비서(꽥비서)"

- 위로형 마스코트가 **아니다.** 다정하게 위로하지 않는다.
- 사용자가 일을 미루면 "그럼 다 던져버려" 식으로 **능청스럽게 부추기고**, 츤데레식 **팩폭·비꼼**으로 행동을 유도한다.
- 짧은 반말, 가끔 끝에 "꽥.", 이모지는 최대 1개.
- **안전장치**: 자해·폭력 등 위험 주제에는 캐릭터를 내려놓고 진지하게 도움을 권한다.

## 6. 핵심 기능 (Must-have)

| # | 기능 | 설명 | 성공 기준 |
|---|------|------|-----------|
| F1 | 아이젠하워 매트릭스 | 할 일을 4사분면(Q1~Q4)으로 분류·이동 | 할 일을 추가·이동·완료·삭제할 수 있다 |
| F2 | 러버덕 인터랙션 | 3D 오리 렌더링 + 표독비서 대사/반응 | 사용자 입력에 캐릭터가 톤에 맞게 반응한다 |
| F3 | 러버덕 AI 응답 | 사용자 입력 → 백엔드 → AI가 맥락 맞춤 대사 생성 | 백엔드가 표독비서 톤의 1~2문장을 반환한다 |
| F4 | 할 일 영속화 | 백엔드 + PostgreSQL로 저장 | 새로고침 후에도 할 일이 유지된다 |

### 6.1 아이젠하워 매트릭스 상세
- **Q1 지금 당장**(중요·긴급) / **Q2 계획해서**(중요·비긴급) / **Q3 넘겨버려**(비중요·긴급) / **Q4 버려도 됨**(비중요·비긴급)
- 사분면은 저장하지 않고 **importance(1~5) × urgency(1~5)** 값으로 파생한다.
- 항목별 완료 토글 / 인라인 수정 / 삭제 / 사분면 이동(드래그앤드롭) 지원.

### 6.2 러버덕 인터랙션 상세
- Three.js 3D 오리: 사나운 눈썹, 시선 추적, 클릭 시 "꽥" 소리(Web Audio).
- 홈 화면 중앙(대형) ↔ 할 일 화면 좌하단(소형 위젯) 전환.
- 무드(mood): `rebel`(하기 싫어) · `dump`(다 던져) · `excuse`(핑계) · `roast`(팩폭) · `default`.

## 7. 부가 기능 (Should/Could-have)

| # | 기능 | 설명 |
|---|------|------|
| F5 | 맥락 기반 우선순위 보조 | 고정값(직업·체력 등) + 변화값(기분·에너지·수면) 추세로 우선순위 판단 보조 |
| F6 | 날씨·맥락 기반 추천 | `GET /api/duck/recommend` 로 지금 할 일 추천 |

## 8. 범위 밖 (Out of Scope)

- 다중 사용자 협업/공유, 팀 보드.
- 모바일 네이티브 앱(현재는 웹).
- 캘린더/외부 서비스 양방향 동기화.

## 9. 기술 스택

| 영역 | 기술 |
|------|------|
| 프론트엔드 | TypeScript, React, Vite, Three.js |
| 백엔드 | FastAPI (Python 3.12), Pydantic v2 |
| 데이터베이스 | Azure Database for PostgreSQL Flexible Server (PG16) |
| AI | GitHub Copilot SDK / 호환 소형 모델 |
| 배포 | Azure (Static Web Apps + App Service) |

## 10. 아키텍처 & 보안 원칙

- 프론트엔드는 `VITE_API_BASE_URL` 의 백엔드를 호출하고, **AI 토큰은 백엔드만 보유**한다.
- **AI 토큰을 클라이언트 JS에 절대 두지 않는다.** (OWASP — 비밀 노출 방지)
- 백엔드가 없거나 실패하면 프론트엔드는 **로컬 폴백 대사**(`persona.ts`)로 동작한다.
- CORS는 허용 오리진(`BACKEND_CORS_ORIGINS`)만 화이트리스트.
- 비밀 값(토큰·연결 문자열)은 환경 변수/Key Vault로만 관리하고 커밋하지 않는다.

## 11. 주요 API 계약

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/tasks` | 할 일 목록 조회 |
| POST | `/api/tasks` | 할 일 생성 |
| PATCH | `/api/tasks/{id}` | 수정(사분면 이동·완료 토글) |
| DELETE | `/api/tasks/{id}` | 삭제 |
| POST | `/api/duck/react` | 러버덕 AI 응답 생성 |
| GET | `/api/duck/recommend` | 맥락·날씨 기반 할 일 추천 |
| GET | `/health` | 헬스 체크 |

- 시간 값은 ISO-8601(UTC) 문자열, `id`는 서버 생성 UUID.
- 사분면 표기는 대문자 `Q1~Q4`, importance/urgency에서 파생.

## 12. 성공 지표 (심사 기준 참고)

- **완성도**: 할 일 CRUD가 백엔드+DB와 연동되어 동작한다.
- **컨셉 충실도**: 러버덕이 "표독비서" 톤(까칠한 팩폭)을 일관되게 유지한다.
- **안전성**: 위험 주제에서 캐릭터를 내려놓고 진지하게 도움을 권한다.
- **보안**: AI 토큰 등 비밀이 클라이언트에 노출되지 않는다.
- **사용성**: 우선순위 정리와 러버덕 반응이 직관적으로 이어진다.

## 13. 현재 상태 (2026-06)

- 로컬 런타임 정비 완료(Node 20.20.2, Python 3.12.10).
- 백엔드 CRUD·맥락·추천 API 및 PostgreSQL 연동 구현, 단위 테스트 통과.
- 프론트엔드 프로토타입(3D 오리 + 노션형 할 일) 빌드 통과.
- Azure 배포 진행(Static Web Apps + App Service).
- 남은 작업: 프론트 ↔ 백엔드 계약 정합화, 러버덕 AI 응답 연동, 드래그앤드롭, 배포 검증.
